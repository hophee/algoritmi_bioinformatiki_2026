# [BA2E](https://rosalind.info/problems/ba2e/) — GreedyMotifSearch with Pseudocounts

## Принцип алгоритма

*GreedyMotifSearch* решает задачу поиска мотивов жадным способом: перебирает все k-меры
первой строки Dna в качестве «якорного» мотива, затем для каждой следующей строки
выбирает наиболее вероятный k-мер по текущему профилю. Ключевое улучшение по сравнению
с базовой версией (BA2D) — использование псевдосчётов (правило Лапласа), которое
исключает нулевые вероятности в профиле и делает алгоритм устойчивее к редким нуклеотидам.

Алгоритм детерминирован: при одинаковых входных данных всегда возвращает один и тот же
результат. Это его главное ограничение — он может застрять в локальном оптимуме.

## Функции

### `hamming_dist`
Стандартная функция из других скрпитов, считающая расстояние Хэмминга.

### `pr_kmer`
Вычисляет вероятность k-мера по профильной матрице как произведение элементов профиля
по всем позициям: $Pr = Profile[nuc_0][0] × Profile[nuc_1][1] × ... × Profile[nuc_{k-1}][k-1]$.
- **Вход**: `kmer` — строка длины k; `profile` — словарь {нуклеотид: список вероятностей}
- **Выход**: `float` — вероятность ∈ [0, 1]

### `profile_most_probable_kmer`
Перебирает все k-меры строки `text` скользящим окном, вычисляет вероятность каждого через
`pr_kmer`, возвращает первый с максимальной вероятностью. Инициализация `best_prob = -1`
гарантирует выбор первого k-мера даже при нулевых вероятностях.
- **Вход**: строка, целое число, профиль
- **Выход**: `str` — один k-мер

### `profile_with_pseudocounts`
Строит профильную матрицу с псевдосчётами (правило наследования Лапласа):
добавляет 1 к каждому нуклеотиду в каждой позиции count-матрицы,
знаменатель = t + 4 (вместо t). Это предотвращает зависимость жадного агоритма от удачного подбора профиля, удаляя нулевые вероятности в произведении вероятностей букв в строке. 
- **Вход**: `list[str]` — список из t мотивов длины k
- **Выход**: `dict[str, list[float]]` — профиль размером 4×k

### `score`
Подсчитывает суммарное количество «непопулярных» нуклеотидов по всем столбцам
мотивной матрицы: для каждого столбца находит максимальную частоту нуклеотида
и добавляет t − max_count к итоговому счёту.
- **Вход**: `list[str]` — список мотивов
- **Выход**: `int` — значение $score \in [0, t \cdot k]$

### `greedy_motif_search`
Основной алгоритм. Внешний цикл перебирает все (n−k+1) k-меров первой строки.
Внутренний цикл по t−1 строкам: строит профиль из уже выбранных мотивов,
выбирает наиболее вероятный k-мер следующей строки.
- **Вход**: список строк ДНК, длина k-мера, количество строк t
- **Выход**: `list[str]` — список из t лучших мотивов
- **Сложность**: $O((n−k+1) \cdot t \cdot n \cdot k)$

# [BA2F](https://rosalind.info/problems/ba2f/) — RandomizedMotifSearch

## Принцип алгоритма

В отличие от жадного поиска, алгоритм начинает со случайно выбранных k-меров из каждой
строки ДНК. Затем итерационно строит профиль по текущим мотивам, заменяет каждый мотив
наиболее вероятным k-мером в своей строке. Цикл продолжается до тех пор, пока счёт
улучшается; при остановке улучшений возвращает BestMotifs.

Так как алгоритм может сходиться к разным локальным оптимумам, он запускается 1000 раз
и возвращается глобально лучший результат. Ключевое отличие от GreedyMotifSearch —
**все t мотивов обновляются одновременно** на каждой итерации.

## Функции

`hamming_dist`, `pr_kmer`, `profile_most_probable_kmer`, `profile_with_pseudocounts`, `score` - уже использовались ранее

### `randomized_motif_search`
Один запуск алгоритма:
1. Случайно выбирает k-мер из каждой строки (`random.randint` задаёт стартовую позицию)
2. Цикл `while True`: строит профиль $\rightarrow$ обновляет все мотивы $\rightarrow$ проверяет score
3. Если score не улучшился — возвращает `best_motifs`

- **Вход**: список строк ДНК, k, t
- **Выход**: `list[str]` — найденные мотивы за один запуск
- **Сложность одной итерации**: $O(t \cdot n \cdot k)$
- **Число итераций**: не фиксировано, в среднем сходится быстро

### Внешний цикл (1000 запусков)
```python
for _ in range(1000):
    motifs = randomized_motif_search(dna, k, t)
```
Запускает алгоритм 1000 раз, сохраняет глобально лучший результат.
Переменные `best_motifs: list[str]` и `best_score: float` хранят текущий рекорд.

*Комментарий*: неясно, зачем задавать `t`, если число строк ДНК итак известно и без проблем считается `len(seq)`.


# [BA2G](https://rosalind.info/problems/ba2g/) — GibbsSampler

## Принцип алгоритма

*GibbsSampler* — усовершенствованная версия RandomizedMotifSearch. Ключевое отличие: на каждой итерации
обновляется **только один случайный мотив** Motif_i, а не все t сразу.

Процедура одного шага:
1. Случайно выбрать индекс $i \in {0, ..., t−1}$
2. Построить профиль из оставшихся t−1 мотивов (без Motif_i)
3. Заменить $Motif_i$ на **случайно сэмплированный** k-мер i-й строки
   пропорционально вероятностям по профилю — функция $RANDOM(p_1,...,p_n)$
4. Проверить score и обновить BestMotifs при улучшении

Такой подход позволяет более плавно исследовать пространство решений и с меньшей
вероятностью застревать в локальных оптимумах. Запускается 20 раз (меньше, чем `randomized_motif_search`,
так как каждый запуск делает N итераций и глубже исследует пространство).

## Функции

`hamming_dist`, `pr_kmer`, `profile_with_pseudocounts`, `score` - уже использовались ранее

### `profile_randomly_generated_kmer`
Реализует функцию $RANDOM(p_1,...,p_n)$, созадвая взвешенную случайную выборку:
1. Вычисляет вероятность каждого k-мера текста через `pr_kmer`
2. Суммирует все вероятности (`total`)
3. Генерирует случайное число $r \in [0, total)$
4. Обходит k-меры, накапливая `cumulative`, возвращает k-мер, на котором $r \le cumulative$

Если все вероятности равны 0 (крайний случай) — возвращает случайный k-мер через
`random.choice`.

- **Вход**: строка, k, профиль
- **Выход**: `str` — один случайно сэмплированный k-мер

### `gibbs_sampler`
Один запуск алгоритма на N итераций:
1. Случайная инициализация мотивов (`random.randint` для каждой строки)
2. Цикл N раз: выбрать случайный i, пересчитать профиль без i-го мотива,
   сэмплировать новый Motif_i, обновить BestMotifs при улучшении

- **Вход**: список строк ДНК, k, t, число итераций N
- **Выход**: `list[str]` — лучшие мотивы за N итераций
- **Сложность**: $O(N \cdot  (t\cdot k + n\cdot k)) = O(N \cdot  t \cdot  n \cdot  k)$

### Внешний цикл (20 запусков)
```python
for _ in range(20):
    motifs = gibbs_sampler(dna, k, t, N)
```
Запускает алгоритм 20 раз (при N = 200 это 4000 итераций суммарно),
сохраняет лучший результат.
